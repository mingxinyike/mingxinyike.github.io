类之间的关系对应代码中的表现

  • 泛化关系 generalize 继承非抽象类
  • 实现关系 realize 继承抽象类
  • 聚合关系 aggregation 整体由部分构成，但整体和部分并不是强依赖的，整体不存在了，部分依然存在
  • 组合关系 composition 整体由部分构成，整体和部分是强依赖的特殊聚合关系，如果整体不存在，部分也不存在
  • 关联关系 association 静态的天然的结构，强关联关系，代码中关联对象一般是以成员变量的形式实现的。
  • 依赖关系 一个对象在运行期间会用到另一个对象的关系，代码中表现为构造类方法以及类方法的传入参数，除了知道对方外还可使用对方的方法和属性。

增加缓存机制

目的：为了缓解请求量增大对服务器的压力，采用缓存机制能够减少对服务器文件的读写。

属性：

  • request

    • If-Modified-Since：第一次请求时response 中的 Last-Modified
    • If-Nont-Match： 浏览器第一次请求时 response 中的 ETag

  • response

    • Cache-Control: max-age=【秒】 || no-cache || no-store || public || private
    • ETag: 当前资源在服务器的唯一标志符
    • Expires
    • Last-Modified: 告诉浏览器当前资源的最后修改时间

HTTP 缓存策略

  • 本地缓存阶段： 先在本地查找该资源，如果存在，且资源未过期，则使用这一资源，不发送 http 请求到服务器
  • 协商缓存阶段： 如果在本地缓存找到对应资源，但是不知道是否过期，则发一个HTTP请求到服务器，然后服务器判断请求的资源在服务器上是否改动过，没有改动过则返回 304，让浏览器使用本地找到的资源
  • 缓存失败： 当服务器发现请求的资源已经修改过，或这是一个新的请求，服务器则返回该资源的数据，并返回200。如果资源未找到则返回 404

本地缓存阶段

  • Expires：缓存过期的绝对时间GMT，如果设置了max-age，expires会失效
  • Cache-Control：max-age=【秒】

前端存储方式

  • Cookie <4k 且会在同域网络请求中进行传输，消耗网络带宽，大量Cookie会导致请求变慢，所以Cookie只保存与权限有关的用户信息。
  • LocalStorage 存储非敏感的静态数据
  • SeesionStorage 关闭浏览器后会清空

协商缓存阶段

  • Last-Modified === if-modified-since 最后修改时间判断
  • ETag === If-None-Match 文件的唯一符

协商流程：
1.客户端请求一个页面（A）。
2.服务器返回页面A，并在给A加上一个Last-Modified/ETag。
3.客户端展现该页面，并将页面连同Last-Modified/ETag一起缓存。
4.客户再次请求页面A，并将上次请求时服务器返回的Last-Modified/ETag一起传递给服务器。
5.服务器检查该Last-Modified或ETag，并判断出该页面自上次客户端请求之后还未被修改，直接返回响应304和一个空的响应体。

Etag 主要为了解决 Last-Modified 无法解决的一些问题：
1、一些文件也许会周期性的更改，但是他的内容并不改变(仅仅改变的修改时间)，这个时候我们并不希望客户端认为这个文件被修改了，而重新GET；
2、某些文件修改非常频繁，比如在秒以下的时间内进行修改，(比方说1s内修改了N次)，If-Modified-Since能检查到的粒度是s级的，这种修改无法判断(或者说UNIX记录MTIME只能精确到秒)；
3、某些服务器不能精确的得到文件的最后修改时间。

Content-Length：尽管并没有在缓存中明确涉及，Content-Length头部在设置缓存策略时很重要。某些软件如果不提前获知内容的大小以留出足够空间，则会拒绝缓存该内容。

Vary：缓存系统通常使用请求的主机和路径作为存储该资源的键。当判断一个请求是否是请求同样内容时，Vary头部可以被用来提醒缓存系统需要注意另一个附加头部。它通常被用来告诉缓存系统同样注意Accept-Encoding头部，以便缓存系统能够区分压缩和未压缩的内容。

单一职责原则

一个类应该只有一项工作，只有一个改变它的理由

单一指责类型：

  • information holder 存储对象并提供对象信息
  • structurer 维护对象和信息之间的关系
  • service provider 处理工作并提供服务
  • controller 控制决策一系列负责的任务处理
  • coordinator 分配工作
  • interfacer 在系统的各个部分转化信息（或请求）

开放封闭原则

软件实体应该是可以扩展的，但是是不可修改的

利用对象的多态性消除条件分支

每当看到大片的 if 或者 else 就应该考虑是否能够利用对象的多态性来重构它们

放置挂钩

模版中在可能变化的地方放置挂钩函数，挂钩返回结果由具体子类决定。程序就拥有了变化的可能。

使用回调

回调是一种特殊的挂钩。可以把一部分易于变化的逻辑封装在回调函数里，然后把回调函数当作参数传入一个稳定和封闭的函数中。如 AJAX请求后要做的一些事情。

派生类型必须可以替换它的基类型
继承子类型需要在语义上等价于基类型里的期望行为。

采用设备像素进行开发

因为一般设计稿都是最大分辨率进行设计的，即基于设备像素进行设计，所以开发的时候也基于设备像素进行开发是比较方便的。通过 document.documentElement.clientWidth 可以拿到设备像素，设备像素 = css像素（理想适口宽度） * window.devicePixelRatio(设备像素比)
需要 js 动态获取当前设备的设备像素，设置 meta，进行缩放。这时，开发中 css 的像素就是针对设备像素而言的。css中的像素值可以直接根据设计稿来定义啦。

针对不同设备进行缩放

不同设备的具有不同的设备像素，如何将针对 iphone6 的设计稿也同样适用于 iphone5 呢？这里采用 rem 来解决这个问题。1rem 等于 html元素的 fontSize。我们针对不同的设备（iphone5, iphone6 等等）设置不同的fontSize = 设备分辨率的1/10。那么 CSS 中，1rem 就等于屏幕的1/10，就可以针对不同的设备宽度进行缩放了。

这里 ratio 的大小针对设计稿中的设备像素宽度决定，比如，设备像素宽度为750px，radio=7.5， 此时 1rem = 100px，定义宽度时就可以直接将设计稿中元素的宽度除以 100，如果设备像素为1200， radio= 12，1rem = 100px, 比较方便计算。

通常采用媒体查询的方式，媒体查询一般分为 321px以下，321px～400px 之间，400px 以上三种。

NEJ define.js 源码简析

define.js 是 NEJ 提供的一套文件依赖管理系统，功能有点类似于 requirejs。

NEJ.define 的使用

注：define.js 除了文件依赖的管理还有一些其他配置，本文仅讨论依赖文件的加载是如何实现的

页面引入依赖库文件，配置参数通过 define.js 的查询参数输入

NEJ.define 为全局函数，输入参数为 _deps 依赖文件列表，以及 _callback 回调函数，其中依赖文件列表中返回的对象会作为回调函数的参数返回，供 _callback 函数调用。
除了依赖文件返回的对象以外，回调函数还会额外注入四个参数，依次为p、o、f、r，其中：

  • p为输出结果集空间，用于注入到其他执行函数中的内容
  • o为对象实例，即{}，用于处理对象默认值，如 var x = options||o;
  • f为函数实例，返回false，用于处理方法默认值，如 var func = x.onready||f;
  • r为数组实例，即[]，用于处理数组默认值，如 var arr = options.list||r;

MVVM 框架中，前端的视图层和数据层需要双向绑定，最难的一点是如何发现数据发生了变化，主要有三种途径：

存取器的封装

缺点：要对每个数据的读取进行封装

脏检测 （ angular 实现方式）

保存数据的新旧值，某些特定的事件发生后，用这个事件之后的数据去跟之前保存的数据进行比对，如果相同，就不触发界面刷新，否则就刷新。

  1. DOM 事件，譬如用户输入文本，点击按钮等。(ng-click)
  2. XHR 响应事件 ($http)
  3. 浏览器 Location 变更事件 ($location)
  4. Timer事件($timeout, $interval)
  5. 执行 $digest()或$apply()

缺点：效率比较低

localStorage, sessionStorage与cookie的同与异

2016-05-10
Learning

本文主要从存储的有效期和作用域来谈谈 localStorage , sessionStorage 以及 cookie 的同与异

localStorage

有效期：永久的，除非主动删除存储的数据，否则将一直保留在用户的电脑中，永不过期

作用域：限定在文档源，文档源通过协议，主机名和端口来确定，同源的文档间共享同样的localStorage数据，同时localStorage的作用域受到浏览器供应商限制，用firefox存储的数据，通过chrome浏览器是无法获取的。

sessionStorage

有效期：与存储数据的脚本所在的最顶层的窗口或浏览器标签页时一样的，一旦窗口或者标签页被永久关闭了，那么存储数据也被删除

作用域：限定在文档源，除此之外还被限制在窗口中，如果同源的文档渲染在不同的浏览器标签页中，那么它们之间互相拥有的是不同的sessionStorage数据

localStorage和sessionStorage的存储方法

  1. setItem()
  2. getItem()
  3. removeItem()
  4. clear()

cookie

有效期：由属性max-age定义有效期

作用域：cookie的path和domain属性配置

secure属性：cookie默认以不安全的形式传递（普通的，不安全的HTTP连接），但是一旦cookie被标识为安全的，那就只能当浏览器和服务器通过HTTPS或者其他的安全协议连接的时候才能传递它。

直接使用不带任何参数的函数引用进行调用，只能使用默认绑定，this指向全局变量

隐式绑定

调用位置是否有上下文，或者说是否被某个对象包含
function foo( ) { …… }
var obj = { foo: foo; }
obj.foo( )
当函数引用有上下文对象时，隐式绑定规则会把函数调用中的this绑定到这个上下文对象。
但是当 var bar = obj.foo; bar( ) 这时this就指向全局变量 因为这里obj.foo是引用

显式绑定

API 调用的上下文绑定
内置函数中有可选参数context ,通常被称为“上下文”，其作用和bind一样，确保你的回调函数使用指定的this

new绑定

使用new调用foo时，会构造一个新对象并把它绑定到foo( )调用中的this 上

优先级：4 > 3 > 2 > 1
特例： 当call apply bind 传入对象时null或者undefined时，实际应用是默认绑定规则。

小结：

  1. HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset –hard commit_id。

  2. 穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。

  3. 要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。

小结

  1. 查看远程库信息，使用git remote -v；

  2. 本地新建的分支如果不推送到远程，对其他人就是不可见的；

  3. 从本地推送分支，使用git push origin branch-name，如果推送失败，先用git pull抓取远程的新提交；

  4. 在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致；

  5. 建立本地分支和远程分支的关联，使用git branch –set-upstream branch-name origin/branch-name；

  6. 从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。
