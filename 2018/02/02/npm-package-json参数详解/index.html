package.json

name

代表项目的唯一标识
规则：

  • 小于等于 214个字符
  • 不能以. 或者 _ 作为开头
  • 不能有大写字母在里面
  • name 会成为 url 的一部分，故name 不能含有URL 非法字符

description

项目描述

version

在package.json 中最重要的两个属性就是name 和 version。package 的更新同时伴随着version的变化。

keywords

数组，有助于搜索 npm search

homepage

项目主页的url

bugs

项目 issue tracker 的 url，或者是email地址

licence

申明一个license 让人们知道合法的使用途径

people fields: author or contributors

author 是一个人

contributors 是一个数组

files

数组，模块下文件名或者文件夹名

main

主入口文件，应该指向模块根目录下的一个文件，很多模块并不写这个属性。

bin

可以配置一个或多个PATH路径下的可执行模块，如果你的模块只有一个可执行文件，且他的命令名称和模块名称一样，你可以只写一个字符串来完成配置

或者有多个可执行文件需要配置

man

制定一个或通过数组制定一些文件来让Linux下的man命令查找文档地址。

directories

CommonJS 通过directories来制定一些方法描述模块结构，暂时没有什么作用

repository

制定一个代码存放的地址

scripts

存放了项目生命周期各个环节需要执行的命令。key 是生命周期中的事件。value 是要执行的命令。 如 start, stop等

config

设置项目中不怎么变化的配置，如port等

dependencies

配置模块依赖的模块列表
npm install module-name –save 自动加入依赖

devDependencies

开发过程中使用的额外测试或者文档框架
npm install module-name –save-dev 自动加入devDependencies依赖

peerDependencies

插件开发的时候，此模块只能作为插件跑在宿主的某个版本范围下，可以配置 peerDependencies

bundledDependencies

指定发布时会被一起打包的模块

optionalDependencies

一个模块可以被使用，但如果模块找不到或者无法获取时可以继续运行你的项目，可以配置到optionalDependencies 中。

engines

项目运行的node版本范围

os

指定模块只能在什么操作系统上跑

cpu

限制模块只能在某某 cpu 架构下运行

preferGlobal

设置为true 代表这个模块主要安装到全局的命令行应用程序，如果安装在本地会提供警告

private

设置为true，npm将拒绝发布他

publishConfig

这个配置是会在模块发布时用到的一些值的集合。如果你不想模块被默认被标记为最新的，或者默认发布到公共仓库，可以在这里配置tag或仓库地址。
