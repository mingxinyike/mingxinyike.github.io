第一章 关于this

this 隐式传递了一个对象的引用，即对上下文环境的引用
this 实际上是在函数被调用时发生的绑定，它指向什么完全取决于函数在哪里被调用

第二章 this 的全面解析

调用栈：为了到达当前执行的位置所调用的所有函数，可以理解为是函数调用列表
调用位置：函数被调用的位置

this绑定规则
找到调用位置

默认规则

直接使用不带任何参数的函数引用进行调用，只能使用默认绑定，this指向全局变量

隐式绑定

调用位置是否有上下文，或者说是否被某个对象包含
function foo( ) { …… }
var obj = { foo: foo; }
obj.foo( )
当函数引用有上下文对象时，隐式绑定规则会把函数调用中的this绑定到这个上下文对象。
但是当 var bar = obj.foo; bar( ) 这时this就指向全局变量 因为这里obj.foo是引用

显式绑定

API 调用的上下文绑定
内置函数中有可选参数context ,通常被称为“上下文”，其作用和bind一样，确保你的回调函数使用指定的this

new绑定

使用new调用foo时，会构造一个新对象并把它绑定到foo( )调用中的this 上

优先级：4 > 3 > 2 > 1
特例： 当call apply bind 传入对象时null或者undefined时，实际应用是默认绑定规则。

第三章 对象

js 中六种主要类型 ( 语言类型 )：

  1. string
  2. number
  3. boolean
  4. null
  5. undefined
  6. object

内置对象

  1. String
  2. Number
  3. Boolean
  4. Object
  5. Function
  6. Array
  7. Date
  8. RegExp
  9. Error

obj.hasOwnProperty( key ) 属性key是否存在于obj中，不包括prototype链
obj.propertyIsEnumerable( key ) 属性key是否存在于obj中，不包括prototype链，且enumerable: true
Object.keys( obj ) 包含一个数组返回所有可枚举属性 只会查找对象直接包含的属性
Object.getOwnPropertyNames ( obj ) 只会查找对象直接包含的属性

for … in 属性存在于对象及其prototype原型链中

使用 in 查找属性会查找整条原型链，无论是否可枚举

遍历：

  1. forEach ( )会遍历所有的值，忽略函数的返回值
  2. every（）遍历直到返回值为 false
  3. some（）遍历直到返回值为 true

第五章 原型

Object.create( obj ) 创建一个对象，并把这个对象的[[prototype]] 关联到指定对象
普通 prototype链最终都指向内置的Object.prototype

Object.prototype包含：
.toString() .valueOf() .hasOwnProperty() .isPrototypeOf()
