css BEM 命名思想

关注这个问题源于我连续写了三个页面，在想css classname的时候想的头疼，不外乎section section1 section2 里面包裹 list, title, summary。。。感觉是那种写完就不想再看一眼的命名。这对于复杂的，需要不断迭代和维护的大项目肯定是不可行的，我就搜索了下解决方案，然后就了解到 BEM

BEM 是什么

BEM 即

  • Block （块）
  • Element (元素，如 img )
  • Modifier (状态，如 active)

其中 block 和 element 之间用连接，和 modifier 之间用 – 连接。
.navigation
.navigationitem
.navigation__item–active

一般情况下我们这么写css

这样写的坏处使得代码过分依赖HTML结构，不易于继续发展进化，同时会造成 .active 常见的类有冲突的风险。

而使用 BEM 风格命名类名，就能够容易的拆解组件，且具有良好的可移植性。
